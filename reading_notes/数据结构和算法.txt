数据结构和算法

介绍
    数据结构：如何使用内存存储数据
    算法：解决常见问题的方法


数据之间的关系：2个方面来理解
    1、逻辑关系：数据与数据直接的逻辑结果，描述的是数据和数据直接的关系，和计算级的存储没有关系
        集合关系、
        线性关系(1对1)：使用索引把数据串联起来
        树型结构(1对多)：一个数据有多个分支
        图型结构(网状结构、多对多)：任意两个数据之间有直接的联系，任何一个数据都可以
    2、物理关系：
        顺序结构：数据在内存中存储在连续位置
        链式结构：把内存中相互独立的存储区使用一定的结构串联起来


算法的特征：
    输入：0个或者是多个输入
    输出：至少有一个输出
    有穷性：算法执行有限次之后结束，不能出现死循环，每一步的执行都在人可承受的范围内
    确定性：算法中的每一步都有确定的含义，不能出现二义性
    可行性：每一步都是可以执行的，执行有限次后结束


算法的效率：
    程序执行的时间不是判断算法效率的最好方法，原因是不客观（计算机硬件环境+软件环境）
    O 表示算法的执行效率：在规模上或者是数量级上的比较
    分析算法，需要考虑：最好的情况、最坏的情况、平均的情况


基本的操作时间复杂度：
    顺序操作：O(1)+
    循环操作：*
    分支操作多个分支的时间复杂度最大的数据


函数执行的次数：
    1+2+3                O(1) 常数阶
    3*n*2 + 3*n+1        O(n) 线性阶
    3*logn + 2*n +1      O(n) 线性阶
    n**3 +n              O(n**3) 立方阶

    O(1) < O(logn) < O(n) < O(nlogn) < O(n**2) < O(n**3) < O(2**n) < O(n!) < O(n**n)
